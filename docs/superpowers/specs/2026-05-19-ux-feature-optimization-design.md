# 芽目标 UX & 功能优化设计

## 目标

本轮优化涵盖 A（UX 打磨）+ B（功能深度），分 6 项：
1. 空状态改版 — 芽芽熊引导 + 快捷模板卡片
2. 目标列表卡片化 — 成长图标 + 微型进度条 + 连续天数
3. 打卡完成反馈升级 — CSS 植物生长动画 + 熊表情变化
4. 复盘页优化 — 对话式复盘（芽芽熊三连问）
5. 打卡日历热力图 — GitHub 风格月度视图
6. 目标编辑 — 修改名称、天数、任务

## 架构

前端拆分：
```
js/
├── api.js          # 所有 fetch 请求
├── state.js         # 状态管理 + 纯函数
├── views/
│   ├── home.js      # 首页
│   ├── create.js    # 创建目标
│   ├── detail.js    # 成长详情
│   ├── review.js    # 复盘
│   ├── bear.js      # 芽芽熊
│   ├── complete.js  # 打卡完成
│   └── calendar.js  # 日历热力图（新）
└── app.js           # 初始化 + 事件绑定 + 路由
```

无新依赖，ES module 不引入，通过全局 `window.App` 共享状态和函数。

后端新增：
- `PUT /api/goals/:id` — 编辑目标
- `GET /api/checkins?goal_id=&year=&month=` — 查询指定月份的打卡日历

## 数据流

```
state.js (window.App.state)
  → views/*.js 读取 state 渲染 HTML
  → 用户操作 → api.js 请求后端
  → 后端更新 MySQL → 返回 JSON
  → app.js 调用 state 更新 + views 重渲染
```

## 组件设计

### 1. 空状态（home.js）
- 无目标时渲染引导式空状态：芽芽熊大图 + 对话气泡 + 4 个快捷模板
- 模板点击自动填入创建表单第一第二步，跳到第三步
- 有目标时正常渲染

### 2. 目标列表卡片（home.js）
- 每张卡片 3 列：左侧阶段图标、中间名称+任务+进度条、右侧计数+连续天数
- 未打卡目标半透明
- 进度条渐变色（绿→黄→粉）

### 3. 打卡完成反馈（complete.js）
- CSS @keyframes 植物从种子→发芽→长叶动画
- 芽芽熊表情切换（开心跳跳）
- 文案动态：首次/连续/中断各有不同

### 4. 对话式复盘（review.js）
- 芽芽熊三连问，每次只显示一个问题
- 输入后点"继续"，出现下一个问题
- 三步完成后一起提交保存

### 5. 日历热力图（calendar.js）
- 单月视图，7 列（日一二三四五六）
- 打卡过的格子填充绿色（深浅按次数）
- 箭头切换月份
- 所有目标合并显示

### 6. 目标编辑（detail.js, create.js 复用）
- 详情页新增"编辑"按钮
- 点击弹出编辑表单（复用创建表单字段）
- 保存调用 PUT API

## 后端 API 变化

```
PUT /api/goals/:id
  body: { name, days, time, task }
  返回: 更新后的 goal 对象

GET /api/checkins?goal_id=&year=&month=
  返回: [{ date: "2026-05-01", count: 3 }, ...]
  goal_id 可选，不传则查所有目标
```

## 样式

- 新增 CSS 动画：@keyframes sprout-grow, @keyframes bear-bounce
- 日历热力图用 CSS Grid 7 列
- 对话式复盘用 transition 切换问题
- 不改变现有设计变量（颜色、圆角、间距）

## 测试要点

- 空状态：0 目标时显示引导，创建后消失
- 目标卡片：进度条百分比正确，连续天数计算正确
- 打卡反馈：动画播放一次，不重复触发
- 复盘对话：三步骤走完才提交
- 日历：跨月数据正确，无打卡日空白
- 编辑：修改后列表即时更新
- 兼容：移动端所有新增 UI 正常显示
